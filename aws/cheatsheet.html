IA classes = retrieval fee/GB; One Zone = 1 AZ only. Intelligent-Tiering auto-shifts tiers, no retrieval fee. Deep Archive = cheapest, <1×/yr access. Lifecycle rules automate Standard→IA→Glacier transitions + expiration by age.

EBS Volumes

  • gp3 SSD — general purpose; IOPS/throughput set independently of size, flat price
  • io1/io2 SSD — Provisioned IOPS; guaranteed sustained IOPS for latency-sensitive DBs
  • st1 HDD — throughput-optimized; big data/log streaming, sequential
  • sc1 HDD — cold, lowest $/GB, rarely accessed

Neither HDD type (st1/sc1) can be a boot volume.

Load Balancers

  • ALB (L7, HTTP/S) — routes on path/host/header; one ALB, many target groups
  • NLB (L4, TCP/UDP) — extreme throughput, lowest latency, static IP per AZ
  • GWLB (L3) — transparently inserts 3rd-party firewall/IDS appliances

DR Strategies

  • Backup & Restore — nothing runs until declared; cheapest, slowest
  • Pilot Light — data tier live; rest deployed but powered off
  • Warm Standby — full stack running 24/7 at reduced size
  • Multi-site Active-Active — full capacity live in 2+ Regions; ~zero downtime, priciest

RPO = max tolerable data loss (time) · RTO = max tolerable downtime

Security Tools

  • CloudTrail — logs every API call: who, when, what
  • Config — records resource config state over time; flags drift from rules
  • GuardDuty — ML/threat-intel over CloudTrail, VPC Flow, DNS logs; finds compromised creds, cryptomining
  • Macie — ML scan of S3 for PII/sensitive data + public-bucket exposure
  • WAF — inspects HTTP(S) requests; allow/block/rate-limit on IP, header, or pattern (SQLi/XSS)
  • Shield Standard (free, auto) vs Advanced (paid, DDoS response team)

Databases

  • RDS/Aurora (relational) — tables, joins, multi-table transactions, fixed schema
  • Aurora — MySQL/PostgreSQL-compatible; distributed auto-growing storage; Aurora Auto Scaling adds/removes replicas on CPU/connections
  • DynamoDB (NoSQL) — key/value or document, no joins, single-digit-ms at any scale; on-demand or provisioned capacity
  • ElastiCache (Redis/Memcached) — managed cache in front of a DB, µs–ms
  • DAX — DynamoDB-only cache, ms→µs, auto invalidation
  • RDS Proxy — pools/reuses connections; failover in seconds w/o dropping app connections; IAM or Secrets Manager auth
  • Read replica (async) — read scaling; promotable for DR
  • Multi-AZ standby (sync) — failover target only, no read traffic

RDS/Aurora automated backups: retention up to 35 days. DynamoDB PITR: rolling 35-day window. Both also support on-demand/manual snapshots kept until deleted.

VPC & Networking

  • Security Group — stateful, per-resource, allow rules only, return traffic auto-permitted
  • NACL — stateless, per-subnet; numbered rules, first match wins; allow + deny needed both directions
  • NAT Gateway — 1 shared = cheapest but cross-AZ charges + single point of failure; 1 per AZ removes both
  • VPC Peering — free in-AZ traffic, non-transitive (no multi-hop)
  • Transit Gateway — hub-and-spoke; hourly + per-GB; pays off past ~6 VPCs vs. mesh peering
  • Gateway Endpoint — S3 & DynamoDB only, free, route-table based
  • Interface Endpoint (PrivateLink) — other services, hourly + per-GB, ENI-based

Connectivity

  • Site-to-Site VPN — IPsec over public internet; up in minutes–hours; cheap, fast, or backup path
  • Direct Connect — dedicated physical link, bypasses internet; longer lead time; steadier bandwidth/latency
  • CloudFront — edge-caches HTTP(S) content; cuts latency + shields origin
  • Global Accelerator — static anycast IPs; routes TCP/UDP over AWS backbone to nearest healthy endpoint; reroutes in seconds
  • Route 53 — DNS + health checks; routing: weighted, latency-based, geolocation/geoproximity, failover

IAM & Access

  • Identity-based policy — attached to a user/group/role
  • Resource-based policy — attached to the resource (bucket/key policy); grants cross-account access without a role in that account
  • SCP — org/OU ceiling on max permissions; never grants, only narrows what identity/resource policies can allow
  • STS — temp credentials, minutes–hours; backs every IAM role; AssumeRole for cross-account
  • IAM Identity Center — workforce SSO across many accounts + managed apps, one console
  • Cognito user pools = app-user directory/auth/JWT; identity pools = swap identity for temp AWS creds via STS

KMS & ACM

  • KMS — default-deny even for root; 1 key policy per key; auto-rotation 1yr default or on-demand; asymmetric keys = manual rotation only
  • ACM DNS-validated cert — auto-renews unattended if still attached/exported
  • ACM email-validated cert — needs human approval
  • Private-CA-issued cert (IssueCertificate API) — NOT auto-renewed
  • Imported/expired cert — never auto-renewed
  • CloudFront cert — must be requested in us-east-1

Scaling & Decoupling

  • SQS — point-to-point buffer; Standard = at-least-once; FIFO = exactly-once + ordered
  • SNS — pub/sub; one topic fans out to many subscribers at once
  • EventBridge — event-driven trigger source feeding Lambda alongside S3/SQS
  • Step Functions Standard — exactly-once, full history, runs up to 1 yr
  • Step Functions Express — at-least-once, ≤5 min, high throughput
  • Auto Scaling — min/max/desired; target-tracking, step, scheduled, or predictive policies
